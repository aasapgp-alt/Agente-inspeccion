F-01 (Critical) — SSRF en /api/drive/proxy_thumbnail
Evidencia:

python

Copy
# app/routers/drive.py:40-50

@router.get("/proxy_thumbnail")

def proxy_thumbnail(url: str, token: Optional[str] = Query(None), db: sqlite3.Connection = Depends(get_db)):

    if not token:

        raise HTTPException(status_code=401, detail="Token no provisto")

    try:

        import requests

        res = requests.get(url, verify=False, timeout=10)   # ⚠️ SSL off + URL arbitraria

        if res.status_code == 200:

            return Response(content=res.content, media_type="image/jpeg")

        return Response(status_code=res.status_code)
Impacto: Un usuario autenticado (incluso Inspector, el rol más bajo) puede:


Acceder al metadata service de la nube (http://169.254.169.254/latest/meta-data/iam/security-credentials/) y robar credenciales de la cuenta de servicio

Escanear la red interna del servidor (puertos, servicios, bases de datos)

Pivotar a otros servicios backend sin restricción

Bypass de firewalls perimetrales

Remediación:

1.
Eliminar el endpoint o restringirlo a una whitelist de hosts permitidos (solo *.googleusercontent.com, drive.google.com).
2.
Si se mantiene: validar urlparse(url).hostname contra la whitelist, rechazar IPs privadas (RFC 1918), loopback, link-local.
3.
Quitar verify=False (siempre).
4.
Bloquear redirecciones (allow_redirects=False).

Copy
from urllib.parse import urlparse

ALLOWED_HOSTS = {"lh3.googleusercontent.com", "drive.google.com"}


parsed = urlparse(url)

if parsed.hostname not in ALLOWED_HOSTS:

    raise HTTPException(400, "Host no permitido")

# Resolver y bloquear IPs privadas/loopback antes de hacer requests.get

res = requests.get(url, timeout=5, allow_redirects=False)  # SIN verify=False
F-02 (Critical) — SSL bypass unconditional en httplib2
Evidencia:

python

Copy
# app/services/drive_service.py:10-15

import httplib2

orig_init = httplib2.Http.__init__

def new_init(self, *args, **kwargs):

    kwargs['disable_ssl_certificate_validation'] = True

    orig_init(self, *args, **kwargs)

httplib2.Http.__init__ = new_init
Impacto: Se ejecuta al importar el módulo y aplica a todas las llamadas httplib2 del proceso, no solo a Drive. En producción, toda la comunicación con Google Drive es MITM-vulnerable. Un atacante en la misma red puede interceptar y modificar archivos PDF de inspección, fotos, credenciales OAuth, etc.

Remediación:


Quitar el monkeypatch. La flag DISABLE_SSL_VERIFY (F-INFO) ya está disponible para entornos Windows con problemas de certificados, y debe ser opt-in.

Si los certificados de Google dan problemas en Windows, usar certifi.where() correctamente (ya está en línea 17) o distribuir el bundle de CA.

F-03 (High) — Default password admin/admin123
Evidencia: init_db.py siembra un usuario admin con password admin123. Está documentado en el README y en .env.example (ADMIN_INITIAL_PASSWORD).

Impacto: Cualquier despliegue que olvide cambiar el password es inmediatamente compromisable. Es el primer usuario que un atacante prueba.

Remediación:

1.
Nunca crear un password default. Si ADMIN_INITIAL_PASSWORD está vacío, forzar al usuario a setearlo en el primer login (flujo de onboarding obligatorio).
2.
Bloquear el primer login si el password es uno de los defaults conocidos (admin123, password, admin).
3.
Agregar check en init_db.py que rechace correr si ADMIN_INITIAL_PASSWORD == "admin123".
4.
Loguear en auditoría cualquier login con password default.
F-04 (High) — Sin rate limiting en login
Evidencia: app/routers/auth.py:54-93 no tiene throttle, captcha, ni lockout.

Impacto: Brute force de passwords sin freno. Combinado con F-03, un atacante puede probar admin/admin123 a velocidad de red.

Remediación:


Implementar rate limit por IP y por username (ej: slowapi o fastapi-limiter).

Sugerencia: 5 intentos/min por IP, 3 intentos/hora por username, lockout temporal de 15 min.

Loguear todos los intentos fallidos en auditoría (ya se hace) y alertar después de N fallos.

from slowapi import Limiter

@router.post("/login")

def login(...): ...
F-05 (High) — Token JWT en query string + check trivial
Evidencia:

python

Copy
# app/routers/drive.py:60-90

@router.get("/proxy_thumbnail")

def proxy_thumbnail(url: str, token: Optional[str] = Query(None), ...):

    if not token:

        raise HTTPException(status_code=401, detail="Token no provisto")  # ⚠️ Solo verifica presencia


@router.get("/imagen/{file_id}")

def get_imagen(file_id: str, token: Optional[str] = Query(None), ...):

    if not token:

        raise HTTPException(status_code=401, detail="Token no provisto")   # ⚠️ Igual
Impacto:

1.
Token en URL: queda en logs de proxy, browser history, Referer headers, server access logs. Es un patrón OWASP A01:2021 (Broken Access Control).
2.
Check trivial: si token es cualquier string no vacío, pasa. Aunque get_imagen después valida con get_user_from_token, el de proxy_thumbnail no valida nada del token.
Remediación:

1.
Mover el Authorization: Bearer a header (que ya es lo que usa el resto de la API).
2.
Para <img> que no soportan headers, firmar URLs de corta duración (signed URLs, tipo S3 presigned) en vez de pasar el token.
3.
Si el endpoint se mantiene, validar el token igual que el resto de la API.
F-06 (High) — JWT_ALGORITHM configurable via env
Evidencia:

python

Copy
# app/core/config.py:33

JWT_ALGORITHM: str = os.getenv("JWT_ALGORITHM", "HS256")
Y en security.py:100:

python

Copy
algorithms=[settings.JWT_ALGORITHM]   # ⚠️ Lista con un solo algoritmo
Impacto: Si JWT_ALGORITHM se setea a none o a un algoritmo distinto al esperado, abre la puerta a algorithm confusion attacks. Un atacante puede forjar tokens firmados con HS256 cuando se espera RS256 con clave pública como secreto (clásico de JWT).

Remediación:


Hardcodear el algoritmo: algorithms=["HS256"].

Eliminar la env var. No hay razón operativa para cambiarlo.

Impacto: En estos casos, set_clause y updates vienen de whitelist interna (columnas hardcoded), no de input del usuario. Pero el patrón es peligroso — un día alguien agrega una columna desde input y se introduce SQLi sin querer.

F-10 (Medium) — Sin security headers
Evidencia: app/main.py no configura headers de seguridad.

Remediación: Agregar middleware custom o secure headers:

python

Copy
@app.middleware("http")

async def add_security_headers(request, call_next):

    response = await call_next(request)

    response.headers["X-Content-Type-Options"] = "nosniff"

    response.headers["X-Frame-Options"] = "DENY"

    response.headers["Referrer-Policy"] = "strict-origin-when-cross-origin"

    response.headers["Content-Security-Policy"] = "default-src 'self'; img-src 'self' data: https://*.googleusercontent.com; ..."

    response.headers["Strict-Transport-Security"] = "max-age=31536000; includeSubDomains"

    return response
F-11 (Medium) — JWT expira en 24h, sin refresh
Evidencia: JWT_EXPIRATION_HOURS = 24 default.

Impacto: Ventana de 24h si un token se filtra. Sin refresh tokens, no hay revocación temprana posible desde el cliente.

Remediación:


Bajar a 1-2h + implementar refresh tokens.

O implementar token rotation: cada N minutos el cliente pide uno nuevo.

Sliding sessions: extender expiración en cada request legítimo.

F-12 (Medium) — Sin rate limit en endpoints costosos
Evidencia: /api/ia/analizar y /api/drive/sincronizar consumen API quota (Gemini, Drive). No tienen throttle.

Impacto: Abuso o bug puede agotar quota y generar costos. DoS económico.

Remediación: Rate limit por usuario (ej: 20 análisis/hora, 3 sincronizaciones/hora).

F-13 (Low) — print() con datos sensibles
Evidencia:

python

# app/routers/ia.py:213-217

print(">>> RAW RESP_TEXT:", res_text)

Impacto: Respuesta cruda de Gemini puede incluir info de clientes (nombres de equipos, ubicaciones). En un container con stdout redirigido a un agregador de logs, queda expuesta.

Remediación: Reemplazar print por logger.debug() o eliminar.

F-14 (Low) — Hash format custom
Evidencia: pbkdf2:sha256:100000$salt$hash (formato propio).

Impacto: Funcional pero no portable. Si en el futuro quieren migrar a passlib o argon2, hay que escribir un verificador custom.

Remediación: Considerar migrar a argon2id (más seguro, resistente a GPU/ASIC) o al menos a passlib con formato estándar.

F-15 (Low) — Respuesta Gemini completa en print
Ver F-13. La respuesta puede contener info del cliente (nombres de equipos, descripciones de fallas). Es un leak de datos en logs.

F-16 (Info) — init_db() corre en cada startup
Evidencia: app/main.py:75-77 ejecuta init_db() en cada arranque.

Impacto: No es vuln pero es anti-pattern. init_db debería correr una sola vez (en deploy), no en cada restart del server.

Remediación: Mover a un script de deploy / entrypoint.sh. En el server, solo verificar que la BD existe.
